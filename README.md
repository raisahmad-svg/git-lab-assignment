                                                       What I Learned
Through this assignment, I learned how to use Git as a version control system. I gained experience in initializing a repository, staging and committing changes, and connecting a local repository to GitHub. I also learned how to use branching and merging to manage different versions of a project, as well as how to resolve merge conflicts manually. In addition, I understood the importance of pull requests in reviewing and integrating changes in a collaborative environment.
                                                       
                                                       Challenges Faced
One of the main challenges I faced was understanding how branching and merging work, especially when conflicts occur. Resolving merge conflicts was initially confusing because I had to manually edit the file and remove conflict markers. Another challenge was remembering the correct sequence of Git commands and understanding the difference between staged and unstaged changes.

                                                   How Git Helps in Real Projects
Git helps improve software development by allowing developers to track changes and maintain different versions of a project efficiently. It enables multiple developers to work on the same project simultaneously without interfering with each other’s work. Features such as branching, merging, and pull requests support collaboration, improve code quality, and reduce the risk of errors in real-world projects.
